017/**
018 * A simple format parser for String-based configuration or data files where JSON is overkill.
019 * Supports only one type, String, but allows each String to have arbitrary nested levels of
020 * String children as if in sub-lists. You can interpret the Strings however you want, and
021 * quoting each String isn't necessary if they are just one word ("bare words" are allowed).
022 * This stores its items in an inner class, {@link ObTextEntry}, which only has a "primary"
023 * String and may have a List of "associated" ObTextEntry values, each of which must have
024 * their own primary String and which may have their own associated List.
025 * <br>
026 * You can use this like any other List, though it will be contain ObTextEntry objects instead
027 * of Strings directly. This allows you to control whether you want to iterate through a
028 * particular primary String's associated entries, if there are any, or to skip over them and
029 * go to the next String in the current List.
030 * <br>
031 * This implements List of String and is modifiable; it extends the List interface with
032 * {@link #parse(CharSequence)} to read a String in the following format, as well as with some
033 * overloads of add() to add just a String with no associated values. A quirk of how this
034 * implements List is that it only considers the top-level Strings to be part of the List for
035 * length and for {@link #contains(Object)}, and will ignore child strings unless you access
036 * them via the {@link ObTextEntry#associated} List on an entry that has associated entries.
037 * <br>
038 * Format example:
039 * <pre>
040 * hello world
041 * 'how are you today?' [just great thanks]
042 * hooray!
043 *
044 * complexity?
045 * [it is possible [yes this is a good example]
046 * 'escapes like \[\'\] all work'
047 * ]
048 *
049 * comments are allowed // like this
050 * comments can have different forms # like this
051 * // block comments like in c are allowed
052 * / * but because this example is in javadoc, this example is not actually a comment * /
053 * // remove the spaces between each slash and asterisk to make the last line a comment.
054 * /[delimit/or block comments with delimiters/delimit]/
055 *
056 * '''
057 * raw strings (heredocs) look like this normally.
058 *     they permit characters without escapes, ]][][[ \/\/\ ,
059 *     except for triple quotes.
060 *     they keep newlines and indentation intact,
061 * except for up to one newline ignored adjacent to each triple quote.
062 * '''
063 *
064 * [[different[
065 * if you may need triple quotes
066 *     in the raw string, use a different syntax that allows delimiters.
067 * here, the delimiter is '''different''', just to be different.]different]]
068 * </pre>
069 * <br>
070 * Inspired strongly by STOB, https://github.com/igagis/stob-java and https://github.com/igagis/stob , but
071 * no code is shared and the format is slightly different. The main differences are:
072 * <ul>
073 * <li>We use square brackets in place of STOB's curly braces to mark children associated with a string.<li>
074 * <li>ObText supports nested block comments using the syntax {@code /[delimiter/contents/delimiter]/} where
075 * delimiter may be empty but must match on both sides, and contents is the body of the comment.</li>
076 * <li>ObText uses Python-like "heredoc" syntax for raw strings surrounded by triple-apostrophes '''like so'''
077 * with optional initial and final newlines in the raw string ignored. An alternate raw string
078 * syntax is present that allows delimiters, using the syntax {@code [[delimiter[contents]delimiter]]}, where 
079 * again delimiter may be empty and contents is the body of the raw string.</li>
080 * <ul>
081 */

224    /**
225     * Parses the given text (a String or other CharSequence) and appends it into this ObText.
226     * @param text a CharSequence (such as a String) using ObText formatting, as described in this class' JavaDocs
227     * @return this ObText object after appending the parsed text, for chaining
228     */
229    public ObText parse(CharSequence text)
230    {
231        m.setTarget(text);
232        ObTextEntry current = null;
233        List<ObTextEntry> ls = this;
234        IntVLA nesting = new IntVLA(4);
235        nesting.add(-1);
236        int t = -1, depth = 0;
237        while (m.find()) {
238            if (m.isCaptured(stringId)) {
239                ls.add(current = new ObTextEntry(m.group(stringId)));
240                nesting.incr(depth, 1);
241            }
242            else if(m.isCaptured(openId))
243            {
244                if(current == null) throw new UnsupportedOperationException("ObText entries can't have associated items without a primary String.");
245                nesting.add(-1);
246                ls = current.openAssociated();
247                depth++;
248            }
249            else if(m.isCaptured(closeId))
250            {
251                if(nesting.size <= 1) throw new UnsupportedOperationException("Associated item sequences in ObText can't end more times than they start.");
252                nesting.pop();
253                depth--;
254                ls = this;
255                for (int i = 1; i < nesting.size; i++) {
256                    ls = ls.get(nesting.get(i)).associated;
257                }
258                nesting.incr(depth, 1);
259            }
260        }
261        return this;
262    }

300    // Used to generate randomized delimiters using up to 9 non-English letters.
301    // call while assigning your state with randomChars(state += 0x9E3779B97F4A7C15L, myChars)
302    // that assumes you have a 9-element char[] called myChars
303    // as long as z/state is deterministic (i.e. based on a hash), this should be too
304    private static void randomChars(long z, char[] mut)
305    {
306        z = (z ^ (z >>> 30)) * 0xBF58476D1CE4E5B9L;
307        z = (z ^ (z >>> 27)) * 0x94D049BB133111EBL;
308        z ^= (z >>> 31);
309
310        mut[0] = letters[(int)(128 + (z & 127))];
311        mut[1] = letters[(int)(128 + (z >>> 7 & 127))];
312        mut[2] = letters[(int)(128 + (z >>> 14 & 127))];
313        mut[3] = letters[(int)(128 + (z >>> 21 & 127))];
314        mut[4] = letters[(int)(128 + (z >>> 28 & 127))];
315        mut[5] = letters[(int)(128 + (z >>> 35 & 127))];
316        mut[6] = letters[(int)(128 + (z >>> 42 & 127))];
317        mut[7] = letters[(int)(128 + (z >>> 49 & 127))];
318        mut[8] = letters[(int)(128 + (z >>> 56 & 127))];
319    }

330    protected static void appendQuoted(StringBuilder sb, String text, Matcher bareFinder)
331    {
332        if(text == null || text.isEmpty()) {
333            sb.append("''");
334            return;
335        }
336        bareFinder.setTarget(text);
337        if(!bareFinder.find())
338            sb.append(text);
339        else
340        {
341            raw.setTarget(text);
342            if(raw.find()) {
343
344                if (text.contains("'''")) {
345                    long state = CrossHash.hash64(text);
346                    char[] myChars = new char[9];
347                    int count;
348                    do {
349                        randomChars(state += 0x9E3779B97F4A7C15L, myChars);
350                        count = StringKit.containsPart(text, myChars, "]", "]]");
351                    } while (count == 12);
352                    sb.append("[[").append(myChars, 0, count).append("[\n").append(text).append("\n]")
353                            .append(myChars, 0, count).append("]]");
354                } else {
355                    sb.append("'''\n").append(text).append("\n'''");
356                }
357            }
358            else if(!text.contains("'"))
359            {
360                sb.append('\'').append(text).append('\'');
361            }
362            else
363            {
364                if(text.contains("\""))
365                {
366                    if(text.contains("'''"))
367                    {
368                        long state = CrossHash.hash64(text);
369                        char[] myChars = new char[9];
370                        int count;
371                        do
372                        {
373                            randomChars(state += 0x9E3779B97F4A7C15L, myChars);
374                            count = StringKit.containsPart(text, myChars);
375                        }while(count == 9);
376                        sb.append("[[").append(myChars, 0, count).append("[\n").append(text).append("\n]")
377                                .append(myChars, 0, count).append("]]");
378                    }
379                    else
380                    {
381                        sb.append("'''\n").append(text).append("\n'''");
382                    }
383                }
384                else
385                {
386                    sb.append('"').append(text).append('"');
387                }
388            }
389        }
390    }

420    private static void iterate(StringBuilder sb, List<ObTextEntry> it)
421    {
422        int len = it.size();
423        ObTextEntry entry;
424        for (int i = 0; i < len; i++) {
425            appendQuotedObText(sb, (entry = it.get(i)).primary);
426            sb.append('\n');
427            if(entry.hasAssociated())
428            {
429                sb.append("[\n");
430                iterate(sb, entry.associated);
431                sb.append("]\n");
432            }
433        }
434    }

565    /**
566     * Can be used to help reading sequences of Strings with ObText-style quotation marking their boundaries.
567     * This returns a {@link ContentMatcher} object that you must call setTarget on before using it.
568     * The argument(s) to setTarget should be the text that might contain quotes, heredoc-style quotes, or just bare
569     * words. Calling {@link ContentMatcher#find()} will try to find the next String, returning false if there's nothing
570     * left or returning true and advancing the search if a String was found. The String might be a special term in some
571     * cases, like "[" and "]" without quotes being syntax in ObText that don't contain usable Strings. That's why,
572     * after a String was found with find(), you should check {@link ContentMatcher#hasMatch()} to verify that a match
573     * was successful, and if that's true, then you can call {@link ContentMatcher#getMatch()} to get the un-quoted
574     * contents of the next String in the target.
575     * @return a {@link ContentMatcher} that must have one of its setTarget() methods called before it can be used
576     */
577    public static ContentMatcher makeMatcher()
578    {
579        return new ContentMatcher();
580    }
581    /**
582     * Can be used to help reading sequences of Strings with ObText-style quotation marking their boundaries.
583     * This returns a {@link ContentMatcher} object that is already configured to read from {@code text}.
584     * The {@code text} should contain Strings that may be surrounded by quotes, heredoc-style quotes, or just bare
585     * words. Calling {@link ContentMatcher#find()} will try to find the next String, returning false if there's nothing
586     * left or returning true and advancing the search if a String was found. The String might be a special term in some
587     * cases, like "[" and "]" without quotes being syntax in ObText that don't contain usable Strings. That's why,
588     * after a String was found with find(), you should check {@link ContentMatcher#hasMatch()} to verify that a match
589     * was successful, and if that's true, then you can call {@link ContentMatcher#getMatch()} to get the un-quoted
590     * contents of the next String in the target.
591     * @param text the target String that should probably have at least one sub-string that might be quoted
592     * @return a {@link ContentMatcher} that can be used immediately by calling {@link ContentMatcher#find()}
593     */
594    public static ContentMatcher makeMatcher(CharSequence text)
595    {
596        return new ContentMatcher(text);
597    }
598
599    /**
600     * Can be used to help reading sequences of Strings with ObText-style quotation marking their boundaries, but no
601     * comments (which allows some additional characters to be used in bare words, like '#').
602     * This returns a {@link ContentMatcher} object that is already configured to read from {@code text}.
603     * The {@code text} should contain Strings that may be surrounded by quotes, heredoc-style quotes, or just bare
604     * words. Calling {@link ContentMatcher#find()} will try to find the next String, returning false if there's nothing
605     * left or returning true and advancing the search if a String was found. Unlike the ContentMatcher produced by
606     * {@link #makeMatcher(CharSequence)}, you can call {@link ContentMatcher#getMatch()} after any successful call to
607     * {@link ContentMatcher#find()}, which will get the un-quoted contents of the next String in the target.
608     * @param text the target String that should probably have at least one sub-string that might be quoted
609     * @return a {@link ContentMatcher} that can be used immediately by calling {@link ContentMatcher#find()}
610     */
611    public static ContentMatcher makeMatcherNoComments(CharSequence text)
612    {
613        return new ContentMatcher(text, patternRelaxed);
614    }

616    public static class ContentMatcher extends Matcher {
617
618        /**
619         * Constructs a ContentMatcher that will need to have its target set with {@link #setTarget(CharSequence)} or
620         * one of its overloads. The target should contain multiple substrings that may have quotation around them; this
621         * class is meant to skip the quotation in ObText's style.
622         */
623        public ContentMatcher()
624        {
625            super(pattern);
626        }
627
628        /**
629         * Constructs a ContentMatcher that already has its target set to {@code text}.
630         * @param text the CharSequence, such as a String, to find possibly-quoted Strings in.
631         */
632        public ContentMatcher(CharSequence text)
633        {
634            super(pattern, text);
635        }
636        /**
637         * Constructs a ContentMatcher that already has its target set to {@code text} and uses an alternate Pattern.
638         */
639        ContentMatcher(CharSequence text, Pattern altPattern)
640        {
641            super(altPattern, text);
642        }
643
644
645        /**
646         * Supplies a text to search in/match with.
647         * Resets current search position to zero.
648         *
649         * @param text - a data
650         * @see Matcher#setTarget(Matcher, int)
651         * @see Matcher#setTarget(CharSequence, int, int)
652         * @see Matcher#setTarget(char[], int, int)
653         * @see Matcher#setTarget(Reader, int)
654         */
655        @Override
656        public void setTarget(CharSequence text) {
657            super.setTarget(text);
658        }
659
660        /**
661         * Supplies a text to search in/match with, as a part of String.
662         * Resets current search position to zero.
663         *
664         * @param text  - a data source
665         * @param start - where the target starts
666         * @param len   - how long is the target
667         * @see Matcher#setTarget(Matcher, int)
668         * @see Matcher#setTarget(CharSequence)
669         * @see Matcher#setTarget(char[], int, int)
670         * @see Matcher#setTarget(Reader, int)
671         */
672        @Override
673        public void setTarget(CharSequence text, int start, int len) {
674            super.setTarget(text, start, len);
675        }
676
677        /**
678         * Supplies a text to search in/match with, as a part of char array.
679         * Resets current search position to zero.
680         *
681         * @param text  - a data source
682         * @param start - where the target starts
683         * @param len   - how long is the target
684         * @see Matcher#setTarget(Matcher, int)
685         * @see Matcher#setTarget(CharSequence)
686         * @see Matcher#setTarget(CharSequence, int, int)
687         * @see Matcher#setTarget(Reader, int)
688         */
689        @Override
690        public void setTarget(char[] text, int start, int len) {
691            super.setTarget(text, start, len);
692        }
693
694        /**
695         * Returns true if {@link #find()} has returned true and the found text is a usable String (not some syntax).
696         * If this returns true, you can reasonably get a (possibly empty) String using {@link #getMatch()}.
697         * @return true if there is a usable String found that can be obtained with {@link #getMatch()}
698         */
699        public boolean hasMatch()
700        {
701            return isCaptured(stringId);
702        }
703
704        /**
705         * Returns the contents of the latest String successfully found with {@link #find()}, without quotation.
706         * You should typically call {@link #hasMatch()} even if find() has returned true, to ensure there is a valid
707         * String that can be acquired (this will return an empty String if hasMatch() returns false, but an empty
708         * String is also potentially a valid result in a successful match, so it should be distinguished).
709         * @return the contents of the latest String successfully found with {@link #find()}
710         */
711        public String getMatch()
712        {
713            return group(stringId);
714        }
715    }
